Probar e implementar imágenes de Windows

¿Vamos a instalar su imagen de Windows en algunos equipos, verdad?

Acelerando el proceso de instalación

Uno de los requisitos es que todos los equipos tengan una cuenta local llamada "John Doe". Otro requisito es omitir tantas preguntas en la OOBE como sea posible. Todo esto se puede automatizar por completo con un archivo de respuestas desatendidas. Para comenzar a crearlo, puede usar el asistente de creación. Para acceder, vaya a Herramientas y haga clic en Creador de archivos de respuestas desatendidas.

Nota:

Puede ver el siguiente cuadro de diálogo al iniciar esta tarea:

Esto puede ocurrir si no tiene el runtime de escritorio .NET 9, que es un requisito para el programa generador de archivos de respuestas subyacente. Puede proceder de 2 maneras: descargando el runtime de escritorio .NET 9, o dejando que DISMTools descargue la versión autónoma después de hacer clic en Sí.

Tenga en cuenta también que el requisito del runtime de .NET puede cambiar en el futuro si el componente subyacente utiliza una nueva versión.

Luego, puede seguir los pasos del asistente. Eventualmente tendrá un archivo de respuestas desatendidas y llegará a esta pantalla:

En esta pantalla puede realizar más tareas con el asistente, como crear otro archivo de respuestas u abrir su ubicación. La tarea que nos interesa realizar es aplicar el archivo de respuestas. Haga clic en el tercer enlace y luego haga clic en Aceptar en este diálogo:

Finalmente, guarde sus cambios. Ahora, crearemos el archivo ISO.

Crear el archivo ISO

Para iniciar la creación del ISO, vaya a Herramientas y haga clic en Crear imagen de disco...

Nota:

Esta tarea requiere el Windows Assessment and Deployment Kit (ADK). Si DISMTools no lo detecta, se le dará la opción de descargar e instalar automáticamente la versión más reciente.

Esto puede tardar un tiempo, durante el cual verá información de progreso:

Después de la instalación, puede continuar normalmente.

Verá la siguiente ventana:

En esta ventana deberá seguir estos pasos

  1. Elija su imagen de Windows. Puede elegirla usando uno de los 3 botones que encontrará en la parte superior:
    • Examinar... le permitirá especificar una imagen de Windows en cualquier ubicación
    • Seleccionar... le permitirá especificar una imagen de Windows entre todas las imágenes montadas. Verá una lista de la que podrá elegir
    • Usar imagen montada utilizará la imagen en la que está trabajando actualmente
  2. Elija un destino para su archivo ISO
  3. Haga clic en Crear. Tenga en cuenta que, si el archivo ISO de destino existe, se sobrescribirá; y que debe asegurarse de que la imagen de Windows tenga todos sus cambios guardados

Para resumir estos pasos, aquí hay una imagen que los muestra:

Estos son los pasos esenciales. Puede configurar más opciones, como copiar a unidades Ventoy, si lo desea. Pero en este ejemplo continuaremos con los valores predeterminados. Después de hacer clic en Aceptar, aparecerá una nueva ventana. Esta ventana creará el archivo ISO que podrá usar más tarde:

Esto tomará un tiempo, así que siéntase libre de hacer otras cosas. Después de que la creación termine, podrá usar su archivo ISO. Puede:

  • Usar su archivo ISO en una máquina virtual
  • Grabarlo en una unidad USB con Rufus
  • Copiarlo a una unidad Ventoy

Continuaremos instalando el sistema operativo en una máquina virtual.

Observaciones finales

Si tiene un laboratorio de computadoras y quiere implementar esta imagen en varios equipos al mismo tiempo, puede optar por realizar una instalación por red. Aunque este Tour no lo cubrirá, puede aprender cómo hacerlo consultando la documentación de Ayuda.

Nota:

DISMTools actualmente solo soporta Windows Deployment Services, y la guía en la documentación de Ayuda está diseñada para esta solución de despliegue.
